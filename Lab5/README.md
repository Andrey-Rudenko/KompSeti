# Лабораторная работа №5

## Задание

Создать TCP-сервер, который передаёт клиенту по запросу файл, лежащий по указанному пути. Разработать протокол взаимодействия с сервером. Создать TCP-клиент, работающий с данным сервером по протоколу, который запрашивает файл по команде пользователя и сохраняет его на машине пользователя. Язык реализации клиента и сервера — C/C++, программы должны быть кроссплатформенными (Windows, Linux) и не использовать сторонних библиотек.

## Файлы

- [`common.h`](./common.h) — обёртки над сокетами Windows (Winsock2) и Linux (BSD sockets).
- [`server.c`](./server.c) — TCP-сервер.
- [`client.c`](./client.c) — TCP-клиент.
- [`files/test.txt`](./files/test.txt) — тестовый файл.

## Протокол

Текстовый заголовок + бинарные данные поверх TCP, порт 5500.

Запрос: `GET <путь>\n`. Ответ: `OK <размер>\n<данные файла>`, либо `ERR NOT_FOUND\n`, либо `ERR BAD_REQUEST\n`. После ответа сервер закрывает соединение.

## Сборка

```
gcc -o server server.c                          # Linux
gcc -o server.exe server.c -lws2_32              # Windows
```
(аналогично для `client.c`)

## Запуск

```
./server [порт]
./client <ip_сервера> <порт> <путь_на_сервере> <куда_сохранить>
```

## Проверка

Собрано и проверено на Windows (gcc/MinGW) и на Linux (WSL Ubuntu, gcc):

```
$ ./client 127.0.0.1 5500 files/test.txt downloaded.txt
Файл получен: files/test.txt (162 из 162 байт) -> сохранён как downloaded.txt

$ ./client 127.0.0.1 5500 files/nope.txt nope_out.txt
Сервер ответил: ERR NOT_FOUND
```
