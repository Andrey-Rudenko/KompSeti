# Лабораторная работа №5

## Задание

Создать TCP-сервер, который передаёт клиенту по запросу файл, лежащий по указанному пути. Разработать протокол взаимодействия с сервером. Создать TCP-клиент, работающий с данным сервером по протоколу, который запрашивает файл по команде пользователя и сохраняет его на машине пользователя. Язык реализации клиента и сервера — C/C++, программы должны быть кроссплатформенными (Windows, Linux) и не использовать сторонних библиотек.

## Файлы

- [`common.h`](./common.h) — общий заголовок: порт по умолчанию, размер буфера, обёртки над сокетами Windows (Winsock2) и Linux (BSD sockets).
- [`server.c`](./server.c) — TCP-сервер.
- [`client.c`](./client.c) — TCP-клиент.
- [`files/test.txt`](./files/test.txt) — тестовый файл для проверки передачи.

## Протокол взаимодействия

Текстовый заголовок + бинарные данные поверх TCP, порт по умолчанию **5500**.

**Запрос (клиент → сервер):** одна строка
```
GET <путь_к_файлу>\n
```

**Ответ (сервер → клиент):**

- Если файл найден — строка с размером файла, сразу за ней — содержимое файла ровно из такого числа байт:
  ```
  OK <размер_в_байтах>\n
  <бинарные данные файла>
  ```
- Если файл не найден:
  ```
  ERR NOT_FOUND\n
  ```
- Если запрос пришёл в неправильном формате (не `GET <путь>`):
  ```
  ERR BAD_REQUEST\n
  ```

После ответа сервер закрывает соединение — на одно подключение обрабатывается один запрос.

## Сборка

Linux:

```
gcc -o server server.c
gcc -o client client.c
```

Windows (gcc/MinGW):

```
gcc -o server.exe server.c -lws2_32
gcc -o client.exe client.c -lws2_32
```

## Запуск

Сервер (порт по умолчанию 5500, можно указать свой):

```
./server
./server 5501
```

Путь к файлу в запросе — относительно той папки, откуда запущен сервер.

Клиент:

```
./client <ip_сервера> <порт> <путь_к_файлу_на_сервере> <куда_сохранить>
# например
./client 127.0.0.1 5500 files/test.txt test.txt
```

## Проверка

Сервер и клиент собраны и проверены и на Windows (gcc/MinGW), и на Linux (WSL Ubuntu, gcc):

```
$ ./client 127.0.0.1 5500 files/test.txt downloaded.txt
Файл получен: files/test.txt (162 из 162 байт) -> сохранён как downloaded.txt

$ ./client 127.0.0.1 5500 files/nope.txt nope_out.txt
Сервер ответил: ERR NOT_FOUND
```
