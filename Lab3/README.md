# Лабораторная работа №3

## Задание

Настроить 2 виртуальные машины с RouterOS или использовать 2 роутера Mikrotik. Настроить в первом роутере выход в интернет через порт Eth1, мост на порты Eth2-Eth4 c DHCP. Во втором роутере настроить выход в интернет через Eth1, дублирующий выход в интернет через Eth2, мост на порты Eth3 и Eth4 с DHCP. Подключить клиентский компьютер (другую виртуальную машину) ко второму роутеру в порт Eth3, во втором роутере порт Eth1 — к порту Eth2 первого роутера, порт Eth2 — к порту своего основного роутера, имеющего доступ в интернет. Убедиться, что клиентский компьютер получил DHCP адрес и выходит в Интернет. Убедиться, что связь с Интернет у клиента остаётся, даже когда порт Eth1 второго роутера отключён от первого роутера (связь идёт по дублирующей ветке, через основной роутер). Сохранить настройки обоих роутеров и прислать на проверку, либо показать экран.

## Схема сети

```
Основной роутер (интернет)
        |
   ether1 [Router1]
        |
   bridge: ether2-ether4 (DHCP, 192.168.10.0/24)
        |
   ether2 ---- ether1 [Router2]                ether2 [Router2] ---- основной роутер
                            |                                              (резервный путь)
                       bridge: ether3-ether4 (DHCP, 192.168.20.0/24)
                            |
                       ether3 ---- Клиент
```

## Отказоустойчивость

На Router2 два DHCP-клиента с разной дистанцией маршрута: `ether1` — distance=1 (через Router1), `ether2` — distance=2 (через основной роутер). При обрыве линка на `ether1` его маршрут пропадает из таблицы, трафик автоматически уходит через `ether2`.
